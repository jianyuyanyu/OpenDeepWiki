# 快速开始

最快的方式是使用 Docker Compose 运行 OpenDeepWiki。本指南将引导你完成设置。

## 前置条件

- 已安装 [Docker](https://docs.docker.com/get-docker/) 和 [Docker Compose](https://docs.docker.com/compose/install/)
- 从 AI 提供商获得 API 密钥（OpenAI、Azure OpenAI、Anthropic 或兼容端点）

## 设置

<Steps>

<Step>
### 克隆仓库

```bash
git clone https://github.com/AIDotNet/OpenDeepWiki.git
cd OpenDeepWiki
```
</Step>

<Step>
### 配置环境变量

编辑 `compose.yaml` 文件并设置你的 AI 提供商凭证。至少需要配置：

```yaml title="compose.yaml"
services:
  opendeepwiki:
    environment:
      # 数据库（sqlite 是默认值，无需额外设置）
      - DB_TYPE=sqlite
      - CONNECTION_STRING=Data Source=/data/opendeepwiki.db

      # AI 服务 — 用于聊天功能
      - CHAT_API_KEY=your-api-key
      - ENDPOINT=https://api.openai.com/v1
      - CHAT_REQUEST_TYPE=OpenAI

      # Wiki 目录生成
      - WIKI_CATALOG_MODEL=gpt-4o
      - WIKI_CATALOG_ENDPOINT=https://api.openai.com/v1
      - WIKI_CATALOG_API_KEY=your-api-key
      - WIKI_CATALOG_REQUEST_TYPE=OpenAI

      # Wiki 内容生成
      - WIKI_CONTENT_MODEL=gpt-4o
      - WIKI_CONTENT_ENDPOINT=https://api.openai.com/v1
      - WIKI_CONTENT_API_KEY=your-api-key
      - WIKI_CONTENT_REQUEST_TYPE=OpenAI

      # 多语言输出（逗号分隔）
      - WIKI_LANGUAGES=en,zh
```

<Callout type="info">
  你可以为所有三个 AI 配置（聊天、目录、内容）使用相同的 API 密钥。仅在想为每个任务使用不同的模型或提供商时才分开设置。
</Callout>
</Step>

<Step>
### 启动服务

```bash
docker-compose up -d
```

或者如果你已安装 `make`：

```bash
make up
```

这将启动两个容器：
- **opendeepwiki**（后端 API）在端口 `8080`
- **web**（Next.js 前端）在端口 `3000`
</Step>

</Steps>

两个服务都健康后，在浏览器中打开 [http://localhost:3000](http://localhost:3000)。

## 关键环境变量

| 变量 | 描述 | 默认值 |
|---|---|---|
| `DB_TYPE` | 数据库类型：`sqlite` 或 `postgresql` | `sqlite` |
| `CONNECTION_STRING` | 数据库连接字符串 | `Data Source=/data/opendeepwiki.db` |
| `CHAT_API_KEY` | 聊天 AI 提供商的 API 密钥 | — |
| `ENDPOINT` | 聊天 AI 端点 URL | `https://api.openai.com/v1` |
| `CHAT_REQUEST_TYPE` | 提供商类型：`OpenAI`、`AzureOpenAI`、`Anthropic` | `OpenAI` |
| `WIKI_CATALOG_MODEL` | 用于生成 wiki 目录结构的模型 | `gpt-4o` |
| `WIKI_CONTENT_MODEL` | 用于生成 wiki 内容的模型 | `gpt-4o` |
| `WIKI_PARALLEL_COUNT` | 并行 wiki 生成任务数 | `5` |
| `WIKI_LANGUAGES` | 输出的语言代码（逗号分隔） | `en,zh` |
| `JWT_SECRET_KEY` | JWT 签名密钥（生产环境需更改） | 默认密钥 |

<Callout type="warn">
  在生产部署中始终更改 `JWT_SECRET_KEY`。默认密钥仅用于开发。
</Callout>

完整的环境变量参考，请参见 [配置](/docs/configuration/environment-variables)。
